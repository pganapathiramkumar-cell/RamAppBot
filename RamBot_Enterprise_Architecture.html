Architecture Enterprise Design Patterns Implemented
Clean Architecture (Hexagonal)
Domain → Application → Infrastructure → Presentation. All services follow strict layer isolation with dependency inversion.
CQRS — Command Query Responsibility Segregation
Write paths (commands) separated from read paths (queries). Queries use Redis cache; commands publish domain events.
Domain-Driven Design (DDD)
Bounded contexts per service. Rich domain entities with invariants, value objects (AlignmentScore), and domain events.
Repository Pattern + DI
IRepository interfaces with PostgreSQL implementations. All dependencies injected via FastAPI Depends() — no singletons in business logic.
Event-Driven Architecture
Every state transition (goal.activated, skill.deployed) publishes a domain event to RabbitMQ. InMemory fallback for resilience.
AI Orchestrator + Chain Pattern
Parallel asyncio.gather() chains for summary + entity extraction + Mermaid generation. Multi-agent pattern for steer/skill analysis.
MCP — Model Context Protocol
AI tooling integration via MCP standard. Claude Code + local agents interact with services through structured tool-call protocol.
Cache-Aside + Pattern Invalidation
Redis cache with 60s TTL on read queries. On every write, targeted wildcard invalidation (e.g. skill:list:{orgId}:*) keeps data fresh.
